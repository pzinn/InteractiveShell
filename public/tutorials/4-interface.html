Introduction

If you've used previously the ``habanero'' server, you should recognize much of the interface. If not, the welcome tutorial can help. Here we focus on new features:

  • (Ka)TeX output
  • Input greek letters and other unicode symbols with [Escape]start of name[Escape] (also works with ZZ, QQ etc)
  • Static syntax highlighting in editor and input; delimiter matching in input
  • Auto-complete in input: keywords with [Tab], previous input with [RightArrow] at end of line
  • Click on past input (including help input) to make it the current input
  • Click on bar left of a cell to delete/shrink/wrap/run it
  • Press F1 to get help on selection
  • Native support for HTML classes
  • You may jump directly to a particular lesson by adding e.g. #tutorial-4-2 to the URL.
  • In order to work, this web interface uses topLevelMode=WebApp, a special output mode of \(\mac\) adapted to mixing TeX and HTML. If you're nostalgic, you can always revert to topLevelMode=Standard for your usual terminal/ascii output.

Some of these points are developed in the next pages.

Also note that a minimal version of this interface exists. It can be embedded as an iframe into any web page.

KaTeX

By default, all output is processed by the \(\mac\) command texMath and then rendered to the browser using KaTeX:
matrix table(5,5,(i,j)->i*j)

All built-in types have a method texMath (or at least they should!). If you define your own type, say,
MyType = new Type of BasicList
then it is a good idea to define your own method texMath. One way to do that is to write
texMath MyType := x -> texMath expression x;
and then for example
expression MyType := x -> ( if #x===0 then return expression 0; if #x===1 then return expression x#0; expression x#0 + 1 / expression drop(x,1) );
The advantage is that several forms of output can then be written at once; e.g., you can also have
net MyType := x -> net expression x; toString MyType := x -> toString expression x;
Let us try:
A = new MyType from a_0..a_5
net A
toString A

Remark: In future versions of \(\mac\), it may not even be needed to write texMath MyType := x -> texMath expression x and similarly for other outputs. Stay tuned!

Hypertext

This interface has native support for HTML output.

Here is how one can create HTML classes in \(\mac\); they are MarkUpTypes with common ancestor the type Hypertext.

The package Text must be (re)loaded:
needsPackage "Text"

MarkUpTypes are self-initializing:
DIV { BOLD { "hello" }, HR {}, OL {"stuff", "more stuff", LI {"and more"} }, ANCHOR { "href" => "http://macaulay2.com", "A link" } }
Note how LI is optional inside OL or UL.

All HTML items can take options:
IMG { "src"=>"http://www.lpthe.jussieu.fr/~pzinn/M2.jpg", "alt"=>"Melbourne likes M2", "width"=>"200" }
A particularly important one is style:
mychord = UL { LI {"C","style"=>"color:red"}, "E", "G", "style"=>"color:green" }
which is why it has its own function:
style(mychord, "background" => "yellow", "font-size"=>"2em")

Style is ignored in Standard mode (net output):
net mychord

Using style is often preferrable to styling directly with HTML attributes; e.g., in the examples above, substitute
style( SPAN { "hello" }, "font-weight" => "bold" )
style( IMG { "src"=>"http://www.lpthe.jussieu.fr/~pzinn/M2.jpg" }, "width" => "10em" )

VectorGraphics

Another type of HTML item is available: SVG (scalable vector graphics). It can be accessed via the package VectorGraphics.
needsPackage "VectorGraphics"

Some examples:
uglyLogo=gList(Ellipse{(0,0),90,30,"stroke"=>"none",Blur=>0.3, "fill"=>linearGradient{("0%","stop-color:red"),("100%","stop-color:yellow")}}, GraphicsText{(-65,-7),"Macaulay2", FontSize=>25,"stroke"=>"black","stroke-width"=>"0.5%","fill"=>"white"}, AnimMatrix=>translation(0,0,10),SizeY=>12)
The next one mixes KaTeX and HTML output:
rgb={"red","green","blue"}; scan(rgb, x -> globalAssign(value x, Circle{"fill"=>x,"stroke"=>"black",SizeX=>0.8,SizeY=>0.8,Margin=>0})); R=QQ[x_red,x_green,x_blue]; print describe R; res cokernel vars R

To access the underlying SVG object, just write
SVG uglyLogo
style(oo,"background"=>"black")

For more information, consult the help:
viewHelp VectorGraphics

Other packages

For your package to work with this interface, it needs to have either a texMath or html method for all defined types.

Say your type is
MyType = new Type of BasicList
One can define as explained before a texMath method. Let us try instead the html method for our type: (we could define it via expression as we did earlier for other modes of output, but for simplicity we define it directly here)
needsPackage "Text"; html MyType := x -> html ( UL toList x )
Let us test it:
new MyType from {"ha","hi",1-ii,QQ[x]}

Here we show how an existing package which already has HTML output can be easily converted to the new interface:
needsPackage "Graphs"
